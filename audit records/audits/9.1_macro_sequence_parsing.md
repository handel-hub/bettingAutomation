# Audit

## Domain
9.1 Macro Sequence Parsing Safety Audit

## Scope
- `MacroEngine.mjs`
- `AutomationController.mjs` (Boot sequence macro loading)

## Architectural Intent
The system must safely load and validate external JSON macros (like `startup.json`) without blocking the Node.js event loop or crashing the orchestrator if the file is manually edited and corrupted.

## Findings

### Finding 1
**Severity**: High
**Category**: Unconditional Boot Crash via Missing Await
**Location**: `AutomationController.mjs` (Line 307)
**Description**: The orchestrator fails to await the `loadSequence()` promise during the boot macro replay logic.
**Evidence**:
- Code E1 (`AutomationController.mjs`):
```javascript
const sequence = this.macroEngine.loadSequence('startup'); // MISSING AWAIT
if (sequence && master) {
     await this.macroEngine.execute(sequence, [master]);
```
- Code E2 (`MacroEngine.mjs`):
```javascript
async execute(commands, targetBrowsers) {
    const executionPromises = targetBrowsers.map(async (b) => {
        for (const command of commands) { // throws TypeError
```
- Static Analysis E4: Because `loadSequence` is `async`, it returns a Promise. The `if (sequence)` check always evaluates to true. The pending Promise object is then passed into `execute()`, where a synchronous `for...of` loop attempts to iterate over it. Since Promises are not iterables, this throws an immediate `TypeError`.
**Impact**: If `replay_action_sequence` is enabled in the configuration, the orchestrator unconditionally crashes on boot, destroying the master and all slaves immediately.

### Finding 2
**Severity**: Medium
**Category**: Lack of Payload Schema Validation
**Location**: `MacroEngine.mjs` (Line 24)
**Description**: The parser catches syntax errors, but completely fails to validate structural schema.
**Evidence**:
- Code E1 (`MacroEngine.mjs`):
```javascript
const rawSequence = JSON.parse(await fsPromises.readFile(seqFile, 'utf-8'));
return rawSequence.map(action => new Command({ ... }));
```
- Static Analysis E4: While `JSON.parse` is correctly wrapped in a `try/catch` and uses non-blocking `fsPromises`, the system blindly assumes `rawSequence` is an Array. If a user manually edits `startup.json` to be a valid JSON object `{ "type": "click" }` instead of an array, `rawSequence.map` throws a TypeError. The system also fails to validate if `action.type` is legitimate or if `action.payload` contains malicious data before passing it to the `Command` constructor.
**Impact**: Corrupt macros will silently fail to execute (returning null due to the catch block) but risk polluting downstream command queues if partially valid data bypasses `Command` generation.

## Overall Assessment
The asynchronous file I/O safely protects the event loop. However, poor type discipline (missing awaits) and complete lack of structural schema validation make the macro engine extremely brittle. The orchestrator is currently fundamentally broken if the startup macro feature is enabled.

## Recommendations
1. **Fix Await**: Add the missing `await` to line 307 of `AutomationController.mjs`: `const sequence = await this.macroEngine.loadSequence('startup');`
2. **Implement Schema Validation**: In `MacroEngine.loadSequence()`, verify that `Array.isArray(rawSequence)` is true before calling `.map()`. Use a validation library (like Zod or Joi) to strictly enforce the `Command` schema.

## Conclusion
The file reading is safe, but the invocation and structural parsing are highly unsafe. The missing await guarantees an unhandled rejection / TypeError at runtime if the feature is activated.
