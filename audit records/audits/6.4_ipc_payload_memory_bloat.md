# Audit

## Domain
6.4 IPC Payload Memory Bloat Audit

## Scope
- `ActionDispatcher.mjs` (Event Payloads)
- `FeatureExtractor.mjs`
- `DataAttributeStrategy.mjs`

## Architectural Intent
The IPC payloads transmitted from the V8 browser isolate to the Node.js master process must be strictly bounded in size. Because the Node.js `ActionDispatcher` retains the last 1,000 actions in memory (`this.actions.push()`), large payloads will multiplicatively consume heap space, leading to fatal Out-Of-Memory (OOM) crashes during long-running recording sessions.

## Findings

### Finding 1
**Severity**: High
**Category**: Unbounded Data Attribute Serialization
**Location**: `DataAttributeStrategy.mjs` (Line 7)
**Description**: The blueprint asks: "Are there any unbounded strings or arrays attached to the `dispatchExecutionEvent` payload?"
**Evidence**:
- Code E1: 
```javascript
locator: '[' + attr + '="' + CSS.escape(val) + '"]',
```
- Static Analysis E4: In modern web applications (like React or Vue SPAs), data attributes (`data-action`, `data-id`) are frequently used to store serialized JSON state or even base64 encoded data. If a targeted DOM element has a `data-action` attribute containing 5MB of stringified JSON, the `DataAttributeStrategy` will construct a 5MB CSS selector string. This string is serialized into `payload.locators`, transmitted over the CDP websocket, and stored in the Node.js `this.actions` array. 
**Impact**: 1,000 recorded actions with 5MB data attributes will consume 5 Gigabytes of Node.js RAM, instantly triggering the `FATAL ERROR: Ineffective mark-compacts near heap limit Allocation failed` crash.

### Finding 2
**Severity**: High
**Category**: Unbounded Input Values
**Location**: `ActionDispatcher.mjs` (Line 412)
**Description**: The `value` property of an `INPUT` event is captured directly from `e.target.value` and attached to the IPC payload.
**Evidence**:
- Code E1:
```javascript
this.inputData.value = e.target.value;
// ...
payload.value = data.value;
```
- Static Analysis E4: If a user (or a script running concurrently) pastes a massive block of text into a `<textarea>`, the entire multi-megabyte string is transmitted across the IPC boundary and permanently retained in the master node's memory sequence array.
**Impact**: OOM crashes if users interact with large text inputs.

### Finding 3
**Severity**: Medium
**Category**: Unbounded Array Growth (Pointer Path)
**Location**: `ActionDispatcher.mjs` (Line 272)
**Description**: The `DRAG` interaction records the exact mouse path.
**Evidence**:
- Code E1:
```javascript
if (dist > 5) { this.pointerData.path.push({x: e.clientX, y: e.clientY}); }
```
- Static Analysis E4: If the browser lags or the user continuously wiggles the mouse over a draggable element for a prolonged period, the `pointerData.path` array grows infinitely. When the mouse is finally released (`mouseup`), this massive array is attached to the `payload.path` property.
**Impact**: CDP websocket latency spikes and unnecessary memory retention.

## Overall Assessment
The recording pipeline lacks crucial payload sanitization and length bounds at the generation layer. By naively trusting the target DOM's attribute lengths and user input volumes, the system invites catastrophic memory leaks in the orchestration process.

## Recommendations
1. **Truncate Data Attributes**: In `DataAttributeStrategy.mjs`, impose a strict maximum length (e.g., 255 characters) on the `val` before constructing the CSS selector.
2. **Truncate Text Input**: In `ActionDispatcher.mjs`, cap `payload.value` to a reasonable maximum (e.g., 10,000 characters) if the goal is only to replay standard form inputs. If the full payload is absolutely required, it should be streamed to a file buffer, not held in the V8 heap array.
3. **Decimate Drag Paths**: Implement point decimation (e.g., Douglas-Peucker algorithm) or limit the maximum length of `pointerData.path` arrays to prevent coordinate bloat.

## Conclusion
The payloads are completely unbounded. The master node is fundamentally exposed to memory exhaustion attacks from the DOM environments it interacts with.
