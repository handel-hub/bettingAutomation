# Audit

## Domain
5.2 Injected Runtime Memory Leak Audit

## Scope
- `ActionDispatcher.mjs` (`InteractionRecognizer`)

## Architectural Intent
The `InteractionRecognizer` operates inside the Chromium isolate (via injected JS). It maintains state machines for pointers, scrolling, and keyboard inputs to aggregate rapid events into semantic BAAOS commands. Because it runs perpetually within the target page, its memory footprint must remain bounded, and it must not prevent V8 from garbage collecting detached DOM nodes.

## Findings

### Finding 1
**Severity**: High
**Category**: Unbounded Array Growth (Memory Leak)
**Location**: `InteractionRecognizer.processPointerEvent()` (Line 268-275)
**Description**: The `pointerData.path` and `pointerData.consumed` arrays grow unbounded if a pointer interaction gets stuck in the `DRAGGING` state.
**Evidence**:
- Code E1: 
```javascript
if (this.pointerState === 'DRAGGING') {
    // ... distance calculation
    if (dist > 5) {
        this.pointerData.path.push({x: e.clientX, y: e.clientY});
    }
}
```
- Static Analysis E4: If a user drags an element out of the browser window and releases the mouse, the browser will not fire a `pointerup` event on the document. The state machine remains permanently in `DRAGGING`. If the user moves the mouse back into the window without clicking, every `mousemove` will satisfy `dist > 5` and push a new coordinate object into `pointerData.path`. Over a long session, this array will accumulate hundreds of thousands of objects, bloating the V8 heap and eventually crashing the page with an Out-of-Memory exception.
**Impact**: Silent memory bloat leading to master browser crash, terminating the entire automation orchestration.
**Root Cause**: Lack of array bounding and lack of state-machine timeout resets for `DRAGGING`.
**Recommendation**: Implement a maximum length for `pointerData.path` (e.g., `if (path.length > 500) path.shift()`). Additionally, implement a "drag idle timeout" that forcibly calls `flushPointer()` if no `mousemove` occurs for X seconds while in `DRAGGING` state.

### Finding 2
**Severity**: Medium
**Category**: Detached DOM Node Leaks
**Location**: `InteractionRecognizer.processPointerEvent()` (Lines 251, 328)
**Description**: The state machine holds a hard reference to `e.target` in `this.pointerData.startTarget`. If the state machine gets stuck (as described in Finding 1), this reference prevents garbage collection.
**Evidence**: 
- Code E1: `this.pointerData.startTarget = e.target;`
- Static Analysis E4: If a user clicks or drags a modal window, and that modal is subsequently closed (removed from the DOM) by the application logic, the `startTarget` retains a JavaScript reference to the removed DOM node. V8 cannot garbage collect the modal or any of its child nodes until the user performs another `mousedown` that calls `flushPointer()`.
**Impact**: Detached DOM trees will accumulate in memory if users frequently click elements that disappear and the pointer state machine fails to flush.
**Root Cause**: Holding strong references to transient DOM nodes in a long-lived singleton.
**Recommendation**: Change `startTarget` to use a `WeakRef(e.target)` instead of a strong reference, or ensure that timeouts rigorously flush the pointer state regardless of missing `pointerup` events.

## Overall Assessment
The injected runtime contains state-machine deadlock vulnerabilities that directly translate into severe memory leaks. Because the code runs inside the Chromium isolate, these leaks bypass Node.js memory limits and directly destabilize the browser tabs.

## Recommendations
1. Enforce array boundaries (e.g., max 500 coordinates) for all dragging paths.
2. Implement safety timeouts for all non-IDLE states to guarantee a return to IDLE (and a flush) even if native browser events are dropped or intercepted.
3. Migrate `target` references inside state data objects to `WeakRef` to ensure the aggregator never blocks DOM garbage collection.

## Conclusion
The `InteractionRecognizer` is currently vulnerable to unbounded array growth and detached DOM node retention if the browser fails to emit terminating pointer events.
