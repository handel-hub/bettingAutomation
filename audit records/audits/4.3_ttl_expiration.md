# Audit

## Domain
4.3 TTL Expiration Accuracy Audit

## Scope
- `ExecutionScheduler.mjs` (`QueueManager.dequeueNext()`)

## Architectural Intent
High-volume, low-priority commands (such as Hovers and Scrolls) become completely irrelevant if their execution is delayed by a synchronization barrier. The TTL logic ensures that if the physical state of the slave browser falls significantly behind the master's capture time, these stale interactions are purged from the queue rather than executed out of context.

## Findings

### Finding 1
**Severity**: Medium
**Category**: Hardcoded Thresholds
**Location**: `ExecutionScheduler.mjs` (Line 121)
**Description**: The Time-To-Live (TTL) expiration window is strictly hardcoded to `1500` milliseconds inline within the `dequeueNext` method. 
**Evidence**:
- Code E1: `if (now - entry.enqueueTime > 1500)`
**Impact**: While 1.5 seconds is a reasonable heuristic for local networks, BAAOS operates distributed slaves across high-latency residential or mobile 4G proxies. A 2-second synchronization delay during a heavy page load on a mobile proxy is normal. Because the TTL is hardcoded, these slaves will silently drop valid scroll/hover interactions instead of executing them once the DOM settles.
**Root Cause**: Magic numbers used for timing heuristics instead of a dynamically configurable property.
**Recommendation**: Extract the `1500` threshold into a central configuration object (e.g., passing `this.settings.ttl` into the `QueueManager`), allowing operators to tune the TTL based on proxy latency profiles.

### Finding 2
**Severity**: None (Positive Observation)
**Category**: Memory Safety & Garbage Collection
**Location**: `ExecutionScheduler.mjs` (Lines 117-124)
**Description**: The blueprint questioned whether expired commands are properly garbage collected. The implementation is highly robust.
**Evidence**: 
- Code E1: The queue array is mutated *before* the TTL check using `const entry = bucket.shift();`.
- Static Analysis E4: If the TTL triggers, it logs and calls `continue`. Because `entry` is declared using block-scoped `const` within the `while` loop, and because `shift()` has already evicted it from the `buckets` reference tree, the object immediately falls out of scope on `continue`. V8's garbage collector will deterministically free the command memory on the next sweep.
**Impact**: The system is completely immune to memory leaks from expired commands.

## Overall Assessment
The expiration logic is mechanically sound and perfectly memory-safe. However, the hardcoded threshold lacks the flexibility required for a distributed infrastructure operating on highly variable network latencies.

## Recommendations
1. Refactor `QueueManager` to accept a `ttlSettings` object upon instantiation, replacing the magic number `1500` with `ttlSettings.aggregatedCommandTTL`.

## Conclusion
The TTL logic cleanly drops stale telemetry and avoids memory leaks, but resolving the hardcoded threshold will make the system resilient to high-latency proxy usage.
