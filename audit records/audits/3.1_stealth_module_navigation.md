# Audit

## Domain
3.1 Stealth Module Navigation Audit

## Scope
- `src/browser/coordination/BrowserLifecycleManager.mjs`
- `src/detection/stealth.mjs`

## Architectural Intent
The Stealth Module must completely obscure headless browser characteristics during navigation, preventing target sites from detecting automated bot behavior via fingerprinting or environment analysis. The orchestrator explicitly emulates a mobile environment (`iPhone 12 Pro`).

## Files Examined
- `BrowserLifecycleManager.mjs`
- `stealth.mjs`

## Execution Model
1. The `BrowserLifecycleManager` initializes `playwright-extra` with `puppeteer-extra-plugin-stealth`.
2. It sets viewport dimensions to match an iPhone.
3. It loads the `iPhone 12 Pro` Playwright device descriptor, applying mobile User-Agents and touch capabilities.
4. If configured, it overrides the `userAgent` using `StealthEngine.getRandomUserAgent()`.
5. It applies custom Canvas spoofing logic.
6. It injects a manual `addInitScript` to forcefully override `navigator.platform` and `window.screen` metrics to match an iPhone, explicitly attempting to "defeat" the stealth plugin's default desktop spoofing.

## Findings

### Finding 1
**Severity**: Critical
**Category**: Fingerprint Detection / WAF Evasion
**Location**: `BrowserLifecycleManager.mjs` & `stealth.mjs`
**Description**: The configuration generates a mathematically impossible, highly unique hardware fingerprint. When `randomize_user_agent` is enabled, the `StealthEngine` returns a desktop User-Agent (e.g., `Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/121.0.0.0`). The `BrowserLifecycleManager` applies this desktop UA, but immediately injects a script forcing `navigator.platform` to `'iPhone'`, `maxTouchPoints` to `5`, and screen dimensions to 390x844. 
**Evidence**:
- Code E1: `stealth.mjs` lines 38-41 hardcode Windows and Mac User-Agents.
- Code E1: `BrowserLifecycleManager.mjs` lines 83-93 force `platform`, `touchPoints`, and screen widths to mobile form factors.
- Runtime Behavior E3: When the browser sends an HTTP request, the `User-Agent` header transmitted to the remote server advertises `Windows NT 10.0; Win64`. However, when the server responds with a JavaScript bot-challenge payload, the evaluation of `navigator.platform` returns `iPhone`, and `ontouchstart in window` returns `true`. This creates a catastrophic divergence between the HTTP layer and the JavaScript execution environment.
**Impact**: Immediate and permanent IP/account flagging upon the first frame load. Modern anti-bot services (Datadome, Akamai, Cloudflare) will instantly drop the session.
**Root Cause**: Incomplete and conflicting override logic mixing desktop UAs with mobile viewports.
**Recommendation**: The `getRandomUserAgent()` method MUST return mobile User-Agents (specifically iOS Safari/Chrome variants) that perfectly match the `iPhone` platform strings. Do not mix desktop UAs with mobile emulation.
**Confidence**: Very High

### Finding 2
**Severity**: High
**Category**: Fingerprint Detection
**Location**: `stealth.mjs`
**Description**: The custom Canvas spoofing script is fundamentally incomplete. It successfully adds noise to the `getImageData` method, but completely ignores `toDataURL` and `toBlob`. Modern canvas fingerprinting scripts draw complex geometries and extract the result using `canvas.toDataURL()` to generate a hash.
**Evidence**:
- Code E1: `stealth.mjs` lines 48-64 hook `originalGetContext` and wrap `ctx.getImageData`. There are no hooks for `toDataURL` or `toBlob`.
- Static Analysis E4: Because `toDataURL` invokes the native C++ rendering pipeline of Chromium without passing through the JavaScript `getImageData` wrapper, the resulting base64 string will perfectly reflect the underlying, un-spoofed hardware fingerprint of the host server.
**Impact**: All spawned slave browsers will share the exact same raw hardware Canvas fingerprint. Anti-bot engines will group these sessions and ban the entire cluster simultaneously for distributed botting.
**Root Cause**: Flawed understanding of Canvas fingerprinting vectors.
**Recommendation**: Extend the Canvas spoofing script to intercept and lightly mutate the pixel data during `toDataURL` and `toBlob` calls, or rely on a hardened stealth plugin specifically designed for complete Canvas spoofing.
**Confidence**: Very High

### Finding 3
**Severity**: Medium
**Category**: Execution Consistency
**Location**: `BrowserLifecycleManager.mjs`
**Description**: The orchestrator engages in a "tug-of-war" with `puppeteer-extra-plugin-stealth`. The stealth plugin natively hooks into Playwright's CDP to inject its own `addScriptToEvaluateOnNewDocument` scripts, which dynamically align `navigator.platform` and `navigator.vendor` with the provided User-Agent. The `BrowserLifecycleManager` then attempts to manually overwrite `navigator.platform` in a separate `addInitScript` block to "defeat" the plugin's desktop spoofing.
**Evidence**:
- Code E1: `BrowserLifecycleManager.mjs` lines 82-83 state `// Defeat Stealth Plugin's desktop spoofing by enforcing mobile properties`.
**Impact**: Playwright executes `addInitScript` blocks in the order they are registered. If `playwright-extra` registers the stealth plugin's scripts *after* the manual `addInitScript` block (or vice versa), the environment variables will flip-flop unpredictably depending on internal plugin timing. If the stealth plugin wins, the browser will report a desktop platform on a mobile viewport.
**Root Cause**: Applying a monolithic desktop stealth plugin to a forced-mobile environment.
**Recommendation**: Configure `puppeteer-extra-plugin-stealth` directly to avoid injecting its User-Agent override sub-plugin. Disable the sub-plugin (`stealth.enabledEvasions.delete('user-agent-override')`), and manage the platform properties manually, ensuring no race conditions exist between competing scripts.
**Confidence**: High

### Finding 4
**Severity**: High
**Category**: Fingerprint Detection / Proxy Leakage
**Location**: `BrowserLifecycleManager.mjs`
**Description**: The `match_proxy_timezone` logic explicitly lies about matching the proxy. Instead of dynamically resolving the proxy IP's geographic timezone, it statically hardcodes the timezone to `Europe/London`. 
**Evidence**:
- Code E1: `BrowserLifecycleManager.mjs` lines 68-70 state `if (this.stealthEngine && this.stealthEngine.settings.match_proxy_timezone === 'true') { contextOptions.timezoneId = 'Europe/London'; }`
- Static Analysis E4: If a proxy is located in New York, its IP geolocation resolves to `America/New_York`. However, the browser's JavaScript environment (`Intl.DateTimeFormat().resolvedOptions().timeZone` and `new Date().getTimezoneOffset()`) will report `Europe/London`. 
**Impact**: This timezone/IP mismatch is a primary heuristic for identifying botnets hiding behind residential proxies. Advanced WAFs will instantly block the request due to the geographical inconsistency.
**Root Cause**: Hardcoded placeholder logic intended for dynamic geographic resolution.
**Recommendation**: Integrate a GeoIP database or API to resolve the proxy IP to a valid IANA timezone string prior to context creation, and dynamically inject that string into `contextOptions.timezoneId`.
**Confidence**: Very High

## Positive Observations
- The use of `playwright-extra` correctly bridges the Puppeteer stealth plugin to Playwright, ensuring that evasion scripts are injected safely via CDP before the main page scripts execute, mitigating the Architectural Risk specified in the blueprint.

## Risks
- **RISK-NEW-11**: Impossible OS/Platform fingerprint combination resulting in instant flagging.
- **RISK-NEW-12**: Incomplete Canvas spoofing exposing raw hardware fingerprints via `toDataURL`.
- **RISK-NEW-13**: Race conditions between custom `addInitScript` overrides and native stealth plugin injections.
- **RISK-NEW-14**: Timezone mismatch between Proxy IP Geolocation and Browser Timezone API.

## Architectural Weaknesses
The system treats stealth as a collection of superficial scripts rather than a cohesive, mathematically sound browser profile. It attempts to bolt a desktop evasion suite onto a mobile emulator, creating contradictory signatures, and uses hardcoded placeholders for critical proxy sync mechanisms.

## Production Readiness
Not ready. The contradictory User-Agent, platform spoofing, and timezone mismatches act as a homing beacon for anti-bot systems.

## Overall Assessment
While the injection timing (via CDP) is architecturally sound, the payload content is fatally flawed. The orchestrator produces a highly visible, contradictory fingerprint that guarantees detection.

## Recommendations
1. Rewrite `StealthEngine.getRandomUserAgent()` to supply only valid iOS/iPhone User-Agents.
2. Patch `HTMLCanvasElement.prototype.toDataURL` and `toBlob` to inject noise.
3. Explicitly disable the `user-agent-override` evasion within the stealth plugin to prevent overriding the custom mobile `addInitScript` definitions.
4. Dynamically resolve Proxy IP timezones instead of hardcoding `Europe/London`.

## Conclusion
The Stealth Module fails to obscure the browser. It instead generates an impossible fingerprint. The inconsistencies must be resolved to achieve evasive capability.
