# Audit

## Domain
5.4 Shadow DOM Penetration Robustness Audit

## Scope
- `ActionDispatcher.mjs` (`InteractionRecognizer.processPointerEvent()`)
- `LocatorIntelligenceEngine.mjs`
- `PipelineContext.mjs`

## Architectural Intent
Modern web applications heavily utilize Web Components and Shadow DOM (e.g., Salesforce Lightning Web Components, Google Lit) to encapsulate UI logic. To reliably replicate user interactions across distributed browsers, BAAOS must be able to pierce Shadow Roots, identifying the specific internal element the user interacted with, rather than just the generic `<my-custom-widget>` shadow host.

## Findings

### Finding 1
**Severity**: Critical
**Category**: Event Retargeting Blindspot (Shadow DOM Omission)
**Location**: `ActionDispatcher.mjs` (Line 251) and `PipelineContext.mjs` (Line 2)
**Description**: The event interception layer fails to utilize `e.composedPath()`, relying exclusively on `e.target` to identify the interacted element. By browser specification, when an event bubbles out of a Shadow Root, `e.target` is "retargeted" to point to the Shadow Host element, hiding the internal DOM structure.
**Evidence**:
- Code E1: `ActionDispatcher.mjs` sets `this.pointerData.startTarget = e.target;` and passes this target into `engine.process(data.target)`.
- Static Analysis E4: Because `LocatorIntelligenceEngine` only receives the retargeted DOM `Element` and never the original `Event` object, it is mathematically impossible for the engine to inspect `e.composedPath()` to discover the true target. The engine mistakenly generates a locator for the generic Shadow Host. When the slave Playwright instance attempts to `.click()` this locator, it clicks the mathematical center of the widget's bounding box, frequently missing the specific internal button the user originally clicked.
**Impact**: Total failure to replicate interactions on any website utilizing Web Components. 
**Root Cause**: Architectural decoupling of the `Event` object from the `LocatorIntelligenceEngine`, combined with a misunderstanding of Shadow DOM event retargeting rules.
**Recommendation**: 
1. Modify `ActionDispatcher.mjs` to capture the true target via `const trueTarget = (e.composedPath && e.composedPath().length > 0) ? e.composedPath()[0] : e.target;`
2. Pass the entire `e.composedPath()` array into the `PipelineContext` so that the `LocatorIntelligenceEngine` can construct a nested `shadowPath` (e.g., `['my-widget', 'my-internal-button']`).

### Finding 2
**Severity**: High
**Category**: Closed Shadow Root Invisibility
**Location**: Native Browser API Limitations
**Description**: The blueprint asks: "How are closed shadow roots handled, if at all?"
**Evidence**: 
- Static Analysis E4: They are completely unhandled. If a shadow root is attached with `{ mode: 'closed' }`, even `e.composedPath()` will not reveal the internal elements; the array stops at the Shadow Host.
**Impact**: Recording fails silently on highly secure or obfuscated target applications utilizing closed shadow roots.
**Recommendation**: To support closed shadow roots, the injected script must override `Element.prototype.attachShadow` at the very top of `debug_injected.js` (before the target application loads) to force `{ mode: 'open' }` or to maintain a weak map of closed shadow roots for internal querying.

## Overall Assessment
The framework's handling of Shadow DOM is functionally non-existent. A comment in `ActionDispatcher.mjs` assumes the engine will magically populate a `shadowPath`, but the engine is architecturally starved of the necessary data (`composedPath`) to do so.

## Recommendations
1. Immediately refactor `ActionDispatcher` to extract the true element using `e.composedPath()[0]`.
2. Pass the `composedPath` array to the Locator Engine to allow the generation of pierced locators (e.g., `>>>` combinator or array-based shadow paths).
3. Implement an `attachShadow` polyfill to force-open closed shadow roots if closed root support is deemed necessary for the target domain.

## Conclusion
The automation platform is currently blind to Web Components. The interaction aggregator must be updated to respect Shadow DOM boundaries.
