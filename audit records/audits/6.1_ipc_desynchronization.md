# Audit

## Domain
6.1 Inter-Process Communication Desynchronization Audit

## Scope
- `ActionDispatcher.mjs` (Injection script: `sendExecution` function)

## Architectural Intent
The `LocatorIntelligenceEngine` runs inside the Chromium V8 isolate and must transmit heavy, complex payloads (containing structural metadata, coordinates, and nested locator strings) back to the Node.js master process over the Chrome DevTools Protocol (CDP) websocket. This IPC channel must be highly resilient against network congestion, Node.js event loop lags, and rapid bursts of user interaction.

## Findings

### Finding 1
**Severity**: High
**Category**: Unbuffered Fire-and-Forget IPC Transmission
**Location**: `ActionDispatcher.mjs` (Line 170) -> `debug_injected.js`
**Description**: The injected script transmits execution payloads to Node.js via `window.dispatchExecutionEvent({ type, payload })`. However, this call is executed immediately on a 1-to-1 basis without any internal queueing, batching, or buffering mechanisms.
**Evidence**:
- Code E1: 
```javascript
function sendExecution(type, payload) {
    if (window.dispatchExecutionEvent) {
        payload.captureTime = Date.now();
        window.dispatchExecutionEvent({ type, payload });
    }
}
```
- Static Analysis E4: A human operator rapidly clicking or scrolling can generate a burst of aggregated events. Because there is no local buffer, `sendExecution` will hammer the CDP websocket with parallel async calls. If the Node.js event loop is temporarily blocked (e.g., due to garbage collection or file I/O) or if the CDP socket experiences backpressure, the `dispatchExecutionEvent` promise may reject or timeout.
**Impact**: Crucial automation commands (like a `CLICK` or `INPUT`) will be permanently lost during high-frequency interactions, silently breaking the replication sequence on the slave browser.

### Finding 2
**Severity**: High
**Category**: Unhandled Promise Rejections on IPC Failure
**Location**: `ActionDispatcher.mjs` (Line 170)
**Description**: Playwright's `exposeBinding` API returns a Promise inside the browser isolate. The `sendExecution` function does not `await` this Promise, nor does it attach a `.catch()` handler.
**Evidence**: 
- Static Analysis E4: If a `V8 Context Destroyed` error occurs (e.g., if the user clicks a button that triggers a rapid page navigation before the IPC transmission fully resolves), the Playwright binding will reject the Promise. Because there is no `.catch()` block, this results in an `Unhandled Promise Rejection` inside the target application's console, which could potentially trigger the target application's own error monitoring (e.g., Sentry, Datadog) and alert the target website administrators to the presence of injected automation scripts.
**Impact**: 
1. The event is permanently dropped.
2. Evasion risk: Unhandled exceptions in the V8 console compromise the stealth of the bot by polluting the target application's error logs.
**Recommendation**: Wrap the IPC call in a `try...catch` (if awaited) or attach a `.catch(() => {})` handler to silently swallow IPC disconnections.

## Overall Assessment
The IPC boundary between the V8 isolate and the Node.js master process lacks standard fault-tolerance mechanisms. It assumes the CDP websocket is infinitely fast, infinitely reliable, and strictly synchronous, which is false for real-world automated browser control.

## Recommendations
1. **Implement an IPC Buffer Queue**: `debug_injected.js` should push payloads into an internal `eventQueue = []`. A debounced flush function should transmit the queue in batches (e.g., `window.dispatchExecutionBatch(events)`).
2. **Implement Retry Logic & Error Swallowing**: If the `exposeBinding` call fails, the batch should remain in the `eventQueue` to be retried on the next tick, and all Promise rejections must be strictly caught and swallowed to prevent bot detection via console telemetry.

## Conclusion
The current IPC transmission logic is extremely brittle. Rapid user interactions will cause dropped payloads and unhandled promise rejections, degrading both replication reliability and evasion stealth.
