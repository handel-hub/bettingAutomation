# Audit

## Domain
8.1 Chromium Isolate Fingerprint Leakage Audit

## Scope
- `BrowserLifecycleManager.mjs`
- `StealthEngine.mjs`

## Architectural Intent
The system orchestrates stealth-configured slave browsers to emulate genuine user hardware and avoid bot detection (Cloudflare, Datadome, Akamai). This requires perfect consistency across HTTP connection layers, browser headers, and internal JavaScript environment variables (navigator, canvas, timezone).

## Files Examined
- `src/browser/coordination/BrowserLifecycleManager.mjs`
- `src/detection/stealth.mjs`

## Findings

### Finding 1
**Severity**: Critical
**Category**: Absolute Fingerprint Contradiction (OS/Hardware)
**Location**: `BrowserLifecycleManager.mjs` (Lines 73-75, 83-84) & `StealthEngine.mjs` (Lines 37-44)
**Description**: The system enforces `navigator.platform = 'iPhone'` via an `addInitScript` hook to spoof a mobile device. However, the `StealthEngine` randomly assigns Desktop HTTP User-Agents (e.g., Windows NT 10.0, Mac OS X). This creates a fatal fingerprint impossibility. 
**Evidence**:
- Runtime Behavior E3: When the browser makes an HTTP request, the TLS payload contains `User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)`. When the target server's anti-bot JS executes `navigator.platform`, the browser's V8 engine returns `iPhone`. The anti-bot backend correlates the HTTP layer with the JS layer, flags the impossible state, and permanently bans the session.
**Impact**: Guaranteed immediate ban from any modern anti-bot system.
**Root Cause**: Lack of dynamic synchronization between the HTTP User-Agent string generation and the JS `navigator` overrides.
**Recommendation**: The `StealthEngine` must parse the assigned `userAgent` and dynamically configure `navigator.platform`, `navigator.userAgent`, and viewport settings to mathematically align with the parsed OS and device type.
**Confidence**: Absolute

### Finding 2
**Severity**: Critical
**Category**: Geolocation / Timezone IP Leak
**Location**: `BrowserLifecycleManager.mjs` (Line 68-70)
**Description**: The application correctly attempts to match the browser's timezone to the assigned Proxy. However, the implementation hardcodes `Europe/London` regardless of the proxy's actual physical location. 
**Evidence**:
- Runtime Behavior E3: A slave is spawned using a US-based proxy (`proxyUrl: http://us-proxy:8080`). The target site receives an HTTP request from a US IP address (GeoIP mapping). The site executes `Intl.DateTimeFormat().resolvedOptions().timeZone` in the browser context. Because line 69 forces `contextOptions.timezoneId = 'Europe/London'`, the browser replies with London time. The massive time discrepancy flags the proxy.
**Impact**: Immediate IP proxy burning and account bans due to proxy detection.
**Root Cause**: Hardcoded literal string where dynamic geographic IP resolution is required.
**Recommendation**: Integrate a lightweight offline GeoIP database or proxy-parsing logic to dynamically map the proxy IP to an IANA timezone string before passing it to `contextOptions.timezoneId`.
**Confidence**: Absolute

### Finding 3
**Severity**: High
**Category**: Canvas Spoofing Detection via `.toString()`
**Location**: `StealthEngine.mjs` (Lines 47-66)
**Description**: The `applyContextStealth` function overrides `HTMLCanvasElement.prototype.getContext` and `ctx.getImageData` to inject random noise into Canvas renders. However, it does not mock the `toString` behavior of these native functions.
**Evidence**:
- Code E1: Lines 49-65 lack any `toString` proxy logic.
- Static Analysis E4: If an anti-bot script executes `HTMLCanvasElement.prototype.getContext.toString()`, a genuine browser returns `function getContext() { [native code] }`. This injected code will return the stringified custom function block, instantly revealing the hook.
**Impact**: Detection of the automation framework's stealth mechanisms.
**Root Cause**: Incomplete JavaScript native function proxying.
**Recommendation**: When overriding native functions, use `Proxy` objects or explicitly overwrite the `.toString` method of the hooked function to return the standard `[native code]` string.
**Confidence**: Very High

## Positive Observations
- The `addInitScript` effectively injects the JavaScript hooks before any document DOM loads, protecting against race conditions.
- Disabling WebRTC hardware decoding and enforcing public interface handling effectively seals WebRTC IP leaks.

## Overall Assessment
The stealth architecture fails fundamentally. The HTTP/TCP fingerprints and the JavaScript execution environments contradict each other massively (Desktop UA + Mobile Platform, Proxy IP + Hardcoded London Timezone, English Locale). The current state is highly toxic to target accounts.

## Recommendations
1. Implement a unified Device Profile Generator that locks UA, Platform, Screen Size, and Touch Points together.
2. Implement dynamic proxy IP-to-Timezone mapping.
3. Fix native function `toString()` leaks.

## Conclusion
The BAAOS architecture currently leaks massive fingerprint anomalies and is unfit for production evasion.
